# Day 2 — Prompt Engineering

## What I worked on
Explored prompt engineering concepts using Claude. Generated two visual explainers
comparing different levels of prompt specification.

## Key learnings
- A vague prompt ("explain prompt engineering") produces a broad but shallow result.
- A detailed specification (defining techniques, workflow, dos/don'ts, design guidelines)
  produces a structured, reusable reference.
- Prompt engineering is itself a skill that benefits from the same principles it teaches:
  be specific, give context, specify format, and iterate.

## Prompts used
1. "Create an image explaining Prompt Engineering"
2. The detailed multi-part specification requesting definition, techniques, workflow,
   dos/don'ts, color/typography guidelines, and alt text.

## Observation
The second prompt produced two coordinated diagrams with a clear workflow loop,
five named techniques, and a seven-row dos/don'ts comparison — significantly more
useful than the first single-frame output.
