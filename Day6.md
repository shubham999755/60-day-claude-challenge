# Day 6 — ATS Resume Optimization

## Original Resume

- Built using a non-standard layout with icons, symbols, and decorative characters
- Lacked a Professional Summary section
- Skills section missing key technologies (Node.js, DSA, Responsive Web Design)
- Section headings used special characters incompatible with ATS parsers
- Projects not ordered by technical complexity
- **ATS Score: 38/100**

## Optimized Resume

- Added a keyword-rich Professional Summary targeting web development roles
- Removed all icons, symbols, and non-standard glyphs for clean ATS parsing
- Surfaced Node.js as a standalone backend skill
- Added "Data Structures & Algorithms" and "Responsive Web Design" to Skills
- Reordered projects by complexity (Spotify Clone with Node.js backend leads)
- Standardized section headings to plain uppercase (EDUCATION, SKILLS, PROJECTS)
- Strengthened action verbs: Developed, Engineered, Implemented, Created
- **ATS Score: 82/100**

## Key Learnings

1. ATS parsers cannot read icons, emojis, or special Unicode characters — always use plain text.
2. A Professional Summary at the top dramatically increases keyword match rate.
3. Skills must be explicitly listed — burying them inside project bullets makes them invisible to ATS.
4. Section headings should be simple and standard (EDUCATION, EXPERIENCE, SKILLS, PROJECTS).
5. Action verbs matter — they align with recruiter search terms and improve readability.
6. Ordering projects by technical complexity signals depth of skill to both ATS and human reviewers.
7. Adding missing keywords like "Data Structures & Algorithms" bridges the gap between your actual skills and what recruiters search for.
