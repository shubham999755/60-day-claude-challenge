# Day 7 — My Claude AI Usage Strategy

> **Challenge:** Understand Claude models, effort settings, and build a personalized AI workflow.

---

## 🧠 My Profile

| Field | Details |
|-------|---------|
| **Role** | Student |
| **Primary Activities** | Coding, Resume / Career Prep |
| **Usage Frequency** | Occasional |
| **Output Need** | Coding Help |

---

## 🤖 Recommended Primary Model

### ⭐ Claude Sonnet 4.6

> Smart enough for complex coding problems, fast enough for daily study sessions, and available on the standard plan — perfect for an occasional student user.

---

## 📊 Model Comparison

| Model | Speed | Power | Best For |
|-------|-------|-------|----------|
| **Haiku 4.5** | ⚡ Fastest | Basic | Quick syntax questions, fixing simple bugs, boilerplate code |
| **Sonnet 4.6** | ⚡⚡ Fast | Strong | Debugging, learning concepts, resume bullets, code review |
| **Opus 4.6** | ⚡⚡⚡ Slower | Highest | Complex algorithms, system design, cover letters, capstone projects |

---

## ⚙️ Effort Level Guide

| Level | When to Use |
|-------|-------------|
| **Low** | Simple lookups, syntax checks, quick definitions |
| **Standard** | Most coding tasks, concept explanations, everyday use |
| **High** | Tricky bugs, resume polishing, portfolio project features |
| **Max** | Capstone projects, dream internship cover letter, complex DSA problems |

---

## 🗂️ My Personalized Claude Workflow

| Task | Best Model | Best Effort | Reason |
|------|-----------|-------------|--------|
| Fix a bug / error message | Sonnet 4.6 | Standard | Most bugs need reasoning, not brute force |
| Learn a new concept (e.g. recursion) | Sonnet 4.6 | Standard | Good explanations with analogies and examples |
| Quick syntax question | Haiku 4.5 | Low | No need to over-think a simple lookup |
| Write resume bullet points | Sonnet 4.6 | High | Word choice matters for recruiters |
| Build a portfolio project feature | Sonnet 4.6 | High | Real code deserves careful generation |
| Write cover letter for internship | Opus 4.6 | Max | High stakes, once-in-a-while task — go all in |
| Design a system / architecture | Opus 4.6 | High | Needs deep reasoning about trade-offs |
| Practice coding interview questions | Sonnet 4.6 | Standard | Consistent, thorough explanations of approach |

---

## 🚫 Biggest Mistakes to Avoid

1. **Using Opus for everything** — It's overkill for quick questions. Save it for high-stakes tasks.
2. **Pasting code without context** — Always describe what the code should do and what error you're seeing.
3. **Accepting the first answer blindly** — Ask Claude to explain its reasoning. Understanding the "why" helps in exams and interviews.
4. **Using Low effort for resume tasks** — Career prep has long-term consequences. Use at least High effort for anything a recruiter will read.
5. **Starting a new chat for follow-ups** — Keep the same conversation going so Claude retains context about your code and prior attempts.

---

## 💡 Key Learnings

- **Model selection matters** — Matching the model to the task saves time and gets better results.
- **Effort level is a multiplier** — The same model gives noticeably better outputs on High vs Low effort for complex tasks.
- **Context is everything** — The more specific your prompt (error message, goal, constraints), the better Claude responds.
- **Career tasks deserve more effort** — Resume and cover letter quality directly impacts opportunities; don't rush these with Low effort.
- **Sonnet 4.6 is the sweet spot** for student use — balances speed, quality, and availability on the free plan.

---

## 🏆 Final Recommendation

> **If I could use only ONE model + ONE effort level:**
>
> ### Sonnet 4.6 + Standard Effort
>
> Covers ~85% of my needs as a student — debugging, learning, writing code, and drafting resume bullets. Fast, smart, and available on the free plan. Switch to High effort when polishing career documents, and Opus only for the most important one-time tasks.
